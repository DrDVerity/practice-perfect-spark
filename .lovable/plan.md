
# Get-Started Subscription + Team & Approval Workflow

## 1. Fix "Get a free account" — Plan Picker Overlay

Replace the blank `LoginWall` popup on `CampaignPreview.tsx` with a **PlanPickerDialog** showing three tiers:

| Plan | Price | Notes |
|------|-------|------|
| Free Trial | $0 for 14 days | No card required |
| Single Location | $399 / month | 1 practice |
| Group / Multi-Location | $699 / month | Multiple locations |

Flow:
1. Click *Get a free account* → PlanPickerDialog opens.
2. Select a plan → Google login (existing `LoginWall` reused, selected plan stored in `sessionStorage` as `pending_plan`).
3. After OAuth returns, `promote-prospect` reads `pending_plan` and stores `plan_tier` + `trial_ends_at` on the profile.
4. Update `src/components/archer/Pricing.tsx` to show the same 3 tiers so the marketing page and picker match.

## 2. 14-Day Nurture Funnel for New Subscribers

- New table `subscriber_nurture_emails` (queued log with `send_at`, `sent_at`, `template_key`).
- On successful signup after prospect promotion, seed 5 scheduled emails:
  - Day 0 — Welcome + activation checklist
  - Day 2 — Connect social accounts nudge
  - Day 5 — Case study / ROI proof
  - Day 9 — Feature spotlight + upgrade CTA
  - Day 13 — "Trial ends tomorrow" final upgrade
- New edge function `send-nurture-emails` runs every 15 min via `pg_cron`, using existing `send-transactional-email`.
- Skip remaining sends once the user upgrades past trial.

## 3. Team Grouping by Practice Website URL

- Add `website_url_normalized` (lowercase, stripped of scheme/`www.`/trailing slash/query) to `accounts` and `profiles`, plus `plan_tier` and `trial_ends_at` on `profiles`.
- On signup / profile edit, normalize the entered practice website URL.
- Users whose normalized URL matches an existing account are auto-added as members of that account via `account_members`.
- First user at a URL = `owner`. Subsequent users default to `user` role (admin can promote to `manager`).
- Only `owner` and `manager` roles can approve budgets — enforced via RLS on `campaign_budgets` using `is_account_owner()` and `is_manager()`.
- Non-privileged users can build campaigns; budgeted plans land in `pending_approval` state.

## 4. Budget Approval Notifications

- New `approval_requests` table (`campaign_id`, `requested_by`, `type='budget'|'strategy'`, `status`, `decided_by`, `decided_at`).
- When a campaign gets a budget or a budgeted plan is generated by a non-approver, insert an `approval_requests` row.
- New edge function `notify-approvers` looks up all owners + managers on the account and emails them via `send-transactional-email` with requester, campaign, budget summary, and a deep link to `/campaign/:id`.
- Approve / Deny buttons on the campaign page write back to `approval_requests` and unlock the campaign for publishing.

## 5. Strategic Plan PDF + Email

- New edge function `generate-strategy-pdf` (Deno, `pdf-lib`) renders:
  - Cover (practice, campaign, budget, dates)
  - Executive summary
  - Strategy phases & tactics
  - Budget allocation table (per channel, per addon)
  - Timeline / milestones
  - Approval + edit instructions
- Store PDF in `landing-pages` bucket (or new `campaign-assets`); save signed URL on `campaigns.strategy_pdf_url`.
- When a budgeted campaign's strategic plan is finalized, `notify-approvers` emails owner + manager with the PDF link and Approve / Modify actions.
